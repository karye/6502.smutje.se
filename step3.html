Mål

Koppla in databussen och RWB-signalen. Arduinon agerar nu som fullständig minnesemulator: den läser adressbussen, levererar instruktioner från en intern array när CPU:n läser, och sparar data när CPU:n skriver. Vi laddar ett minimalt testprogram och ser CPU:n exekvera det.

Varning — busskollision! När CPU:n skriver (RWB = LÅG) driver CPU:n databussen. Arduinon måste sätta PORTA till INPUT (högimpedans) innan PHI2 går hög. 100 Ω-motstånden skyddar kretsarna om en kollision ändå skulle inträffa.

Arduino-kod

// Steg 3 — Full minnesemulering med testprogram
#define PHI2  2
#define RESB  4
#define RWB   3

uint8_t memory[65536];  // 64K adressrymd

void setup() {
  Serial.begin(115200);
  pinMode(PHI2, OUTPUT);
  pinMode(RESB, OUTPUT);
  pinMode(RWB,  INPUT);

  DDRK  = 0x00;  // PORTK = INPUT (adressbuss låg)
  DDRF  = 0x00;  // PORTF = INPUT (adressbuss hög)

  // Ladda reset-vektor: startadress $8000
  memory[0xFFFC] = 0x00;  // låg byte
  memory[0xFFFD] = 0x80;  // hög byte → $8000

  // Ladda testprogram på $8000:
  // NOP ($EA) i en loop — CPU:n bara snurrar
  memory[0x8000] = 0xEA;  // NOP

  // ── Reset-sekvens ──
  digitalWrite(RESB, LOW);
  for (int i = 0; i < 5; i++) { clockPulse(); }
  digitalWrite(RESB, HIGH);
  for (int i = 0; i < 10; i++) { clockPulse(); }

  Serial.println("Klar — CPU kör från $8000");
}

void clockPulse() {
  digitalWrite(PHI2, LOW);
  delay(10);

  uint16_t addr = (PINK << 0) | (PINF << 8);
  bool rwb = digitalRead(RWB);

  // Förbered data INNAN PHI2 går hög
  if (rwb == HIGH) {
    // CPU LÄSER — Arduino levererar instruktion/data
    DDRA  = 0xFF;         // PORTA = OUTPUT
    PORTA = memory[addr]; // Lägg ut data från minnesarrayen
  } else {
    // CPU SKRIVER — Arduino läser
    DDRA = 0x00;          // PORTA = INPUT (högimpedans!)
  }

  digitalWrite(PHI2, HIGH);
  delay(10);

  // Om CPU skrev, spara data
  if (rwb == LOW) {
    memory[addr] = PINA;
  }

  // Logga till Serieövervakaren
  Serial.print("$");
  Serial.print(addr, HEX);
  Serial.print(rwb ? " LÄS " : " SKRIV ");
  Serial.println(memory[addr], HEX);
}

void loop() {
  clockPulse();
}

Testning

  1. Ladda upp koden och öppna Serieövervakaren
  2. Du ska se att CPU:n läser $FFFC och $FFFD, hoppar till $8000, och sedan läser $EA (NOP) om och om igen
  3. Databuss-LED:arna visar värdet 11101010 (NOP) — lysdioderna D7,D6,D5,D3,D1 lyser

Förväntat resultat

✅ CPU:n läser reset-vektorn, hoppar till $8000 och exekverar NOP i en loop. Serieövervakaren visar att adressen stannar på $8000 och datavärdet är $EA varje gång. LED-raden visar binärkoden för NOP. CPU:n är nu fullt fungerande med minnesemulering!
