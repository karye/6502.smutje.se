Steg 2 — Adressbuss och reset-sekvens

Mål

Koppla in adressbussen (A0–A15) och RESB. Verifiera att CPU:n efter reset läser startvektorn $FFFC/$FFFD via serieövervakaren.

Nya komponenter

AntalKomponent
17Kopplingstrådar (16 adress + 1 RESB)

Kopplingsschema

Arduino-kod

#define PHI2  2
#define RESB  4

void setup() {
  Serial.begin(115200);
  pinMode(PHI2, OUTPUT);
  pinMode(RESB, OUTPUT);
  DDRK = 0x00;  // PORTK (A8-A15) = INPUT
  DDRF = 0x00;  // PORTF (A0-A7)  = INPUT

  // Reset-sekvens
  digitalWrite(RESB, LOW);
  for (int i = 0; i < 5; i++) { pulse(); }
  digitalWrite(RESB, HIGH);
  for (int i = 0; i < 10; i++) { pulse(); }
  Serial.println("Klar");
}

void pulse() {
  digitalWrite(PHI2, HIGH); delay(10);
  digitalWrite(PHI2, LOW);  delay(10);
}

void loop() {
  digitalWrite(PHI2, HIGH); delay(10);
  uint16_t addr = (PINK << 0) | (PINF << 8);
  Serial.println(addr, HEX);
  digitalWrite(PHI2, LOW); delay(10);
}

Testning

  1. Öppna Serieövervakaren (115200 baud)
  2. Första adresserna efter reset ska vara $FFFC och $FFFD
✅ Förväntat: CPU:n läser reset-vektorn. Adresserna syns i serieövervakaren.
