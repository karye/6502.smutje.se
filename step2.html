Steg 2 — Adressbuss och reset-sekvens

Steg 1 Steg 2 Steg 3 Steg 4 ← Översikt

Mål

Koppla in adressbussen och reset-linjen. Verifiera att CPU:n efter reset läser sin startvektor från $FFFC och $FFFD via serieövervakaren.

Nya komponenter (utöver steg 1)

AntalKomponentSyfte
–16 kopplingstrådarAdressbuss A0–A15 till Arduinos analoga ingångar
–1 kopplingstrådRESB (pin 40) till Arduino D4

Notera den korsade mappningen: CPU A0–A7 → Arduino A8–A15 (PORTK), CPU A8–A15 → Arduino A0–A7 (PORTF). Detta gör att vi kan läsa hela 16-bitarsadressen i två 8-bitars registerläsningar.

Arduino-kod

// Steg 2 — Reset-sekvens + adressbussavläsning
#define PHI2  2
#define RESB  4

void setup() {
  Serial.begin(115200);
  pinMode(PHI2, OUTPUT);
  pinMode(RESB, OUTPUT);

  // Ställ analoga portar som ingångar
  DDRK  = 0x00;  // PORTK (A8-A15) = INPUT
  DDRF  = 0x00;  // PORTF (A0-A7)  = INPUT

  // ── Reset-sekvens enligt datablad ──
  digitalWrite(RESB, LOW);
  for (int i = 0; i < 5; i++) {
    digitalWrite(PHI2, HIGH); delay(10);
    digitalWrite(PHI2, LOW);  delay(10);
  }
  digitalWrite(RESB, HIGH);  // Släpp reset

  // Ge CPU:n 7 klockcykler för reset-sekvensen
  for (int i = 0; i < 10; i++) {
    digitalWrite(PHI2, HIGH); delay(10);
    digitalWrite(PHI2, LOW);  delay(10);
  }

  Serial.println("Reset klar — redo att läsa adressbuss");
}

void loop() {
  digitalWrite(PHI2, HIGH);
  delay(10);

  // Läs 16-bitars adressbuss
  uint16_t addr = (PINK << 0) | (PINF << 8);
  Serial.print("Adress: $");
  Serial.println(addr, HEX);

  digitalWrite(PHI2, LOW);
  delay(10);
}

Testning

  1. Ladda upp koden och öppna Serieövervakaren (115200 baud)
  2. Du ska se en ström av adresser — CPU:n försöker hämta instruktioner
  3. De första adresserna efter reset ska vara $FFFC och $FFFD — reset-vektorn
  4. Därefter hoppar CPU:n till den adress som ligger på $FFFC/$FFFD (med flytande databuss blir värdet slumpmässigt)

Förväntat resultat

✅ Serieövervakaren visar adresser i snabb följd. De första två avläsningarna efter reset är $FFFC och $FFFD. CPU:n är vid liv och försöker exekvera kod — redo för steg 3 där vi ger den riktiga instruktioner via databussen.
